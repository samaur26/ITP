# Python Reading
## Chapter 1
I already knew about operators, numbers, and strings because of our work in JavaScript. I understand that an operator is used for some sort of math operation, and strings are sequences of letters. I also already knew the difference between integers and floating-point numbers, as those concepts have been discussed during our work with JavaScript.

I didn't know that you could simply switch between types of values with functions. For example, the float() function will convert strings with digits and decimals to floating-point numbers. The same goes for a string that's a whole number - the int() function allows you to convert those to integers.

## Chapter 3
I already understood how functions work, and how those functions can be used inside of other functions. Functions are defined, and then called in order to execute them in the code. These functions can be put together and nested inside other functions in order to create a larger function.

What I didn't know was how stack diagrams and frames worked. These diagrams show values of each variable and the function the variable is in, with each function being represented by a frame. Those frames are stacked in a way where you can see which function was called in order, which is nice to keep track of the variables being used.

## Chapter 5
I already knew about boolean expressions and logic, as well as how logical operators like < and > work. I also understand the general principle of using operators with if statements, as this is also a big part of JavaScript.

I understand the difference between chained and nested conditionals more now. If a chained conditional is used, then only 1 of the chains have to be true in order to execute the rest of the statement. If a nested conditional is used, then when 1 condition is filled, it will either go to the nested condition or execute the other condition.
